Wykonaj przegląd copy i UX writing dla projektu/strony.

Zakres minimalny (sprawdź i udokumentuj):
- Headlines: clarity, value proposition, keyword relevance, length.
- CTA (Call-to-Action): specific, action-oriented, visible, compelling.
- Microcopy: buttons, labels, placeholders, error messages, help text.
- Tone of voice: consistency, alignment z target audience, brand personality.
- Clarity: jargon, passive voice, sentence length, readability (Flesch-Kincaid).
- Scannability: bullet points, headings, whitespace, visual hierarchy.
- Error messages: helpful, actionable, empathetic, no blame.
- Forms: field labels, placeholders, inline validation, success messages.
- Navigation: menu labels, breadcrumbs, clear paths, no dead ends.
- Accessibility: plain language, abbreviations explained, link text (nie "kliknij tutaj").

Wynik (format obowiązkowy):
1) Executive summary (5–10 punktów: największe problemy + wpływ na conversion/UX)
2) Lista problemów Must / Should / Could + priorytet + conversion impact
3) Checklista wdrożeniowa (konkretne taski + przykłady lepszego copy)
4) Jeśli to repo: wskazanie plików/fragmentów do zmiany + rewrity (krótkie)
